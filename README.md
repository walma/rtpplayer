# RTP Player

Минимальный Android-проект на Kotlin с использованием `LibVLC` для воспроизведения RTP/UDP-потоков в реальном времени.

## Особенности реализации

- **LibVLC Integration**: Использование нативного движка VLC для декодирования сетевых потоков.
- **Picture-in-Picture (PiP)**: Поддержка режима "картинка-в-картинке" при сворачивании приложения или нажатии кнопки.
- **Авто-восстановление**: Плеер автоматически переподключается к потоку при разворачивании приложения (восстановление Surface).
- **Jetpack Compose**: Современный UI с поддержкой динамических настроек.

## Настройка сетевых параметров (Latency Tuning)

В интерфейсе приложения доступны настройки для минимизации задержки:

| Параметр | Описание | Рекомендуемые значения |
| :--- | :--- | :--- |
| **Network Caching** | Размер сетевого буфера в мс. | **150-300** для локальной сети, **1000+** для интернета. |
| **Clock Jitter** | Сглаживание неравномерности пакетов. | **0** для минимальной задержки, **250+** для плавности на плохом Wi-Fi. |
| **Clock Synchro** | Алгоритм синхронизации видео/аудио. | **0** (Auto), **-1** (Disabled), **1** (Strict). |

## Оптимизация источника (FFmpeg)

Для стабильного воспроизведения без заиканий звука источник должен обеспечивать скорость кодирования `speed=1.000x`.

### Рекомендуемая команда для вещания:
```sh
ffmpeg -re -i input.mp4 \
  -c:v libx264 -preset ultrafast -tune zerolatency \
  -b:v 3500k -maxrate 3500k -bufsize 7000k \
  -c:a aac -b:a 128k \
  -f mpegts "udp://<ANDROID_IP>:5004?pkt_size=1316"
```

### Решение проблем со звуком и задержками:
1. **Заикание звука**: Часто вызвано тем, что CPU не успевает кодировать (`speed < 1.0`). Используйте `-preset ultrafast` или аппаратное ускорение (`-c:v h264_nvenc` для NVIDIA).
2. **Рассыпание картинки**: Снизьте битрейт (`-b:v`) до 3000k-4000k, так как UDP пакеты могут теряться в Wi-Fi при высоком трафике.
3. **Пакетная оптимизация**: Флаг `pkt_size=1316` упаковывает ровно 7 TS-пакетов в один MTU, что стандартно для IPTV и минимизирует потери.

## Примеры URI в приложении

- `udp://@:5004` — Прослушивание входящего Unicast на порту 5004.
- `rtp://@239.0.0.1:5004` — Прием Multicast-группы.

## Как перенести этот модуль в другой проект

1. **Копирование**: Скопируйте папку модуля `rtp-player` в корень нового проекта.
2. **Settings**: Добавьте `include(":rtp-player")` в `settings.gradle.kts` нового проекта.
3. **Library Mode**: Если хотите использовать его как библиотеку, в `rtp-player/build.gradle.kts`:
   - Замените `com.android.application` на `com.android.library`.
   - Удалите `applicationId`.
4. **Зависимости**: В основном модуле нового проекта добавьте `implementation(project(":rtp-player"))`.
5. **Манифест**: Не забудьте скопировать разрешения (INTERNET, ACCESS_WIFI_STATE, CHANGE_WIFI_MULTICAST_STATE) и настройки PiP для Activity в основной `AndroidManifest.xml`.
6. **LibVLC**: Убедитесь, что в новом проекте разрешен доступ к `mavenCentral()`, откуда подкачивается `libvlc-all`.

## Требования

- Android 8.0+ (для PiP режима).
- Wi-Fi сеть без блокировки UDP/Multicast трафика.
- Источник потока в той же подсети.
